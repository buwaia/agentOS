# 从一次 AI 幻觉，到三层治理框架

> 今天用 Claude Code 解数学题，解出了一套东西。

---

## 一、事件

我让 Claude 证明琴生不等式。

它给了我这个：

```
f''(x) = -1/x² < 0，所以 f(x) 是凹函数，
由 Jensen 不等式得 f((a+b)/2) ≥ (f(a)+f(b))/2，
两边取 exp 得结论。∎
```

我说：**你产生了 AI 幻觉，帮改正，用拉马努金的方式给我。**

它重新给了我这个：

```
在直径为 a+b 的半圆里，
从圆上任意一点向直径作垂线，垂线长度 = √(ab)。
垂线不超过半径 (a+b)/2。
几何事实直接给出结论，无需推导。
```

第一个是推导，第二个是洞见。区别不在于对错，在于能不能让人**看见**。

---

## 二、真正的问题

问题不是 Claude 这次说错了。

问题是：**这件事消失在了对话里。下次打开新 session，照犯。**

AI 没有记忆。每次对话都是白板。我踩过的坑、纠正过的错，不会影响下一次。

---

## 三、我搭了什么

### Drishti — 项目知识库自动维护系统

> Drishti：梵文 दृष्टि，直接看见的能力。灵感来自拉马努金——他不记推导，只记洞见。

基于 Claude Code Hook 系统，做两件事：

**Session 开始：** 自动读入四个项目文档注入给 Claude，让它一开口就了解项目背景。

**Session 结束：** 扫描本次对话关键字，把新洞见追加写回文档。

```
知识库结构：
├── PRODUCT.md     产品知识（用户画像、业务规则）
├── PROJECT.md     项目状态（Sprint、阻塞、待办）
├── TECH.md        技术决策（ADR、不可逆选择）
└── IMPROVEMENT.md 改进方向（Tech Debt、禁止事项）
```

三条设计边界：

| 边界 | 设计 |
|------|------|
| 只在 session 结束触发 | 不打断对话 |
| 命中关键字才记录 | 闲聊不写库 |
| 只追加不修改 | 知识只增不减 |

---

### 三层治理 — Principles → Rules → Gates

光有 Drishti 还不够。我还需要一套框架，把"这次 AI 幻觉"蒸馏成可以重复使用的知识。

**Layer 1 Principles（不变的原则）**

```
P1: 洞见优先于推导
    直接看见答案，而不是算出答案。

P2: AI 输出质量是产品底线
    解析质量 = 产品质量，低质量必须被捕获。

P3: 规则有生命周期
    每条规则都应该被自动化取代，成熟后退休。
```

**Layer 2 Rules（人工判断，有生命周期）**

```
R001: 解答必须包含直觉性描述
      → 追溯 P1
      → 过期条件：Gate 稳定 30 天后退休

R002: AI 幻觉信号必须被 Drishti 捕获
      → 追溯 P2
      → 过期条件：Gate 稳定 60 天后退休
```

**Layer 3 Gates（可执行的自动化检查）**

```bash
# 验证解答质量
bash check-answer-quality.sh "你的解答内容"

# 验证关键字捕获覆盖
bash check-drishti-keywords.sh

# 一键跑全部
bash run-all.sh
```

## 四、有治理 vs 没治理

| 维度 | 无治理 | 有三层蒸馏 |
|------|--------|-----------|
| 解答质量 | 机械符号推导 | 直觉图示 + 类比 |
| 问题被捕获 | 用户抱怨后消失 | 关键字命中，哨兵记录 |
| 知识沉淀 | 对话消失，下次照犯 | 写入文档，跨 session 持续 |
| 可验证性 | 无 | 20 个测试用例，一键验证 |
| 自动化升级 | 靠人记 | Gate 稳定后 Rule 自动退休 |

---

## 五、一句话

> 治理的终点不是更多规则，而是规则的消亡。

拉马努金不记推导过程，他的笔记本里只有洞见。

这套系统的设计遵循同样的原则：Principles 永久，Rules 会退休，Gates 会稳定后消亡。

问题消失在对话里，是系统的失败。问题变成文档里的一行改进方向，是系统的成功。
